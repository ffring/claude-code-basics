# Claude для офисного руководителя

Методичка для тех, у кого есть подчинённые, планёрки, таблицы, почта, документы, отчёты руководству и куча параллельных задач — но кто не пишет код.

Цель — сделать так, чтобы Claude закрывал рутину, а вы занимались тем, ради чего вас наняли. Без терминала, без программирования, без долгих настроек.

Курс собран по принципу «постепенное усложнение»: начинаем с обычного браузера, заканчиваем тем, что Claude сам открывает приложения и заполняет таблицы за вас. На каждом шаге — теория и упражнение на ваших реальных задачах.

## Что такое Claude и при чём тут разные продукты

**Claude** — большая языковая модель от компании Anthropic, конкурент ChatGPT и Gemini. Сильные стороны: длинный контекст (помнит весь проект), аккуратная работа с документами, таблицами и кодом, осторожность в выводах (меньше фантазирует).

Anthropic делает несколько продуктов вокруг этой модели — у них одинаковая модель внутри, но разная упаковка и сценарии:

| Продукт | Где живёт | Кому |
|---|---|---|
| **Claude.ai** | Браузер, мобильное приложение | Все. Чат, документы, аналитика, исследования |
| **Claude Desktop** | Mac, Windows | Те, кому нужен Cowork и интеграция с локальными файлами |
| **Cowork** | Внутри Claude Desktop | Тем, кто хочет, чтобы Claude сам делал работу на компьютере (организация файлов, отчёты, рутинные операции) |
| **Claude в Chrome** | Расширение для браузера | Тем, кто заполняет формы, делает заказы, работает с веб-сервисами |
| **Claude Code** | Терминал | Разработчикам. Этой методички не касается, но в этом репо есть отдельный курс |

Эта методичка покрывает первые четыре. Шаги выстроены так, чтобы вы могли остановиться там, где вам хватает: для большинства руководителей хватает первых 5 шагов, шаги 6–10 — для тех, кто хочет глубже.

| Шаг | Что осваиваем | Зачем |
|-----|---------------|-------|
| 1 | Тарифы и установка | Выбираем правильный тариф, ставим приложения |
| 2 | Первый разговор и артефакты | Учимся ставить задачи и получать готовые документы |
| 3 | Файлы и контекст | Загружаем PDF, Excel, Word — Claude разбирает |
| 4 | Projects | Постоянное рабочее пространство для каждой темы |
| 5 | Memory | Claude помнит вас и ваши правила между чатами |
| 6 | Connectors | Подключаем Gmail, Drive, Slack, Notion |
| 7 | Skills | Встроенные «навыки» Claude и свои команды |
| 8 | Cowork | Claude как агент: сам делает работу на компьютере |
| 9 | Plugins | Готовые наборы под роли (продажи, финансы, операционка) |
| 10 | Claude в Chrome | Агент в браузере: заполняет формы, собирает данные с сайтов |

В конце — сквозные сценарии руководителя (утро менеджера, совещание под ключ, отчёт для босса, аналитика, рутина) и блок про безопасность.

## Шаг 1. Тарифы и установка

### Что выбрать

Начать можно с **Free** — Claude бесплатно даёт чат, артефакты, базовую работу с файлами и память. Этого достаточно, чтобы пройти первые три шага и понять, нужна ли вам подписка.

Что включено в каждом тарифе (актуальные цены 2026):

| Тариф | Цена | Кому |
|---|---|---|
| **Free** | $0 | Попробовать. Лимит сообщений, 5 проектов, ограниченный контекст |
| **Pro** | $17/мес (год) или $20/мес | Большинству руководителей. Безлимитные проекты, Cowork, Claude в Chrome, коннекторы, расширенное мышление |
| **Max** | от $100/мес (5×) до $200/мес (20×) | Если упираетесь в лимиты Pro. Приоритетный доступ к новым фичам |
| **Team** | $20–25/место (5+ мест) | Команда: общие проекты, общие скиллы, SSO, без обучения на ваших данных |
| **Enterprise** | от $20/место + индивидуально | Большие компании: SSO, аудит-логи, HIPAA, контекст 500K, контроль данных |

Memory работает на всех тарифах, включая Free (с марта 2026). Cowork — только Pro и выше, и только на десктопе. Claude в Chrome — все платные тарифы.

### Установка

**1. Регистрация.** Заходите на [claude.ai](https://claude.com), регистрируетесь через рабочую почту или Google. Бесплатно, без карты.

**2. Десктоп-приложение** (нужно для Cowork). Скачать на [claude.com/download](https://claude.com/download). Версии для Mac и Windows. Linux — официально нет, через браузер.

**3. Мобильное приложение.** App Store / Google Play, ищите «Claude by Anthropic». Полезно для голосовых задач в дороге и фотографий документов.

**4. Расширение для Chrome** (опционально, для шага 10). [claude.com/chrome](https://claude.com/chrome) или через Chrome Web Store.

### Безопасность с самого начала

- На Free и Pro Anthropic **может** использовать ваши разговоры для обучения — отключается в настройках (Settings → Privacy → Help improve Claude).
- На Team и Enterprise обучение по умолчанию **выключено**.
- Никогда не загружайте: пароли, банковские реквизиты, медицинские данные, нерегулируемые персональные данные сотрудников и клиентов.
- Если у вас есть NDA-документы или коммерческая тайна — Team-тариф минимум.

**Упражнение 1.** Зарегистрируйтесь на claude.ai, поставьте десктоп-приложение, мобильное приложение. Откройте Settings → Privacy и решите, оставлять ли «Help improve Claude» включённым. Если в компании есть NDA — выключите.

## Шаг 2. Первый разговор и артефакты

### Как ставить задачу

Claude отличается от Google не тем, что ищет короткий ответ, а тем, что выполняет задачу. Чтобы он попал в цель с первой попытки, в задаче должно быть четыре вещи:

1. **Роль и контекст.** Кто я и зачем мне это: «Я операционный директор, готовлю презентацию на совет директоров».
2. **Сама задача.** Что нужно сделать: «Сделай слайд про наш Q1».
3. **Формат результата.** В чём нужен ответ: «PowerPoint, 10 слайдов, корпоративный тон, основные KPI цифрами».
4. **Ограничения.** Что не делать: «Без воды, без советов, не выдумывай цифры».

Плохо: «Сделай мне отчёт».
Хорошо: «Я операционный директор. Сделай Excel с финансовыми KPI команды на основе прикреплённого выгруза. 4 таба: P&L, кадры, продажи, маркетинг. Используй формулы для итогов. Без графиков — добавлю сам».

Это не «магический промпт-инжиниринг» — это просто внятная постановка задачи, как подчинённому.

### Что такое артефакты

**Артефакт** — это документ, который Claude создаёт прямо в разговоре, в отдельной панели справа. Это не просто текст в чате. Артефакт можно скачать, отредактировать, переделать.

Что Claude умеет делать в артефактах:
- Документы Word (.docx) — служебки, регламенты, отчёты с форматированием.
- Таблицы Excel (.xlsx) — с формулами, заголовками, форматированием.
- Презентации PowerPoint (.pptx) — со слайдами, маркерами, заголовками.
- PDF — финальные документы для рассылки.
- Интерактивные дашборды на HTML (открываются в браузере, кликабельные графики).
- Диаграммы (mermaid, flowchart, mindmap).
- Письма, шаблоны, контент.

Не нужно просить «сделай в виде артефакта» — Claude сам понимает, когда уместно, и предлагает. Если не предложил — попросите явно: «Сделай это в виде Word-документа».

### Голос и картинки

- На мобильном — голосовой ввод (микрофон). Удобно по дороге: «Запиши, что нужно подготовить ко встрече с N».
- Картинки — фотография доски с совещания, скрин таблицы, диаграмма от партнёра. Claude всё разбирает и может перенести в нормальный документ.

**Упражнение 2.** Сформулируйте по 4-блочной схеме одну реальную задачу из вашей рабочей рутины. Получите результат как артефакт (документ или таблицу). Сравните с тем, как вы обычно делаете эту задачу: сколько времени съедает у вас, что Claude сделал хуже / лучше.

## Шаг 3. Файлы и контекст

### Что Claude умеет читать

Перетащите файл в чат — и спрашивайте про него.

| Тип | Примеры использования |
|---|---|
| PDF | Договор → краткое саммари + риски. Регламент → проверь соответствие нашему |
| Word | Документ → переформулируй формальнее / короче / на английский |
| Excel / CSV | Таблица → найди аномалии, сделай pivot, посчитай KPI |
| PowerPoint | Презентация → выжимка / переделай в наш стиль / переведи |
| Картинки | Скриншот ошибки, фото доски, диаграмма от подрядчика |
| Аудио / видео | Запись звонка → транскрипция → протокол + задачи |

Лимиты на Free и Pro: ~30 МБ на файл, до 20 файлов в одном сообщении. На Max и выше — больше. Фактический предел определяется не размером файла, а его содержимым: Claude может «не влезть» в контекст, если PDF на 500 страниц мелким шрифтом.

### Контекстное окно — что важно знать

«Контекст» — это всё, что Claude держит в памяти в рамках одного разговора: ваши вопросы, его ответы, прикреплённые файлы. У Claude он огромный (200 000–500 000 знаков для Pro/Max), но не бесконечный.

Что это значит на практике:
- Один проект — один разговор. Не мешайте «договор с поставщиком» и «питч для инвесторов» в одном чате.
- Если разговор стал длинным и Claude начинает путаться — нажмите «New chat», сделайте новый чат с тем же файлом.
- Если файлов много и они одни и те же — это уже **Project** (следующий шаг).

### Связка «файл + чёткий запрос»

Шаблон, который работает:

```
Контекст: я [роль], готовлю [что].
Прикреплённый файл: [что это, кратко].
Задача: [что сделать с файлом].
Формат: [как нужен результат].
Что не делать: [ограничения].
```

Пример:
```
Я директор по маркетингу, готовлю еженедельный обзор для CEO.
Прикреплённый файл: выгрузка из Google Analytics за неделю.
Задача: найди 3 главные аномалии (резкие падения / всплески), объясни возможные причины, дай рекомендации.
Формат: короткое письмо в Slack, не длиннее 200 слов.
Не делай: общих советов «надо больше тестировать», только конкретику по этим данным.
```

**Упражнение 3.** Возьмите реальный документ или таблицу со своей работы. Загрузите. Поставьте задачу по шаблону «контекст + задача + формат + ограничения». Проверьте: получилось ли сэкономить время по сравнению с тем, чтобы делать руками.

## Шаг 4. Projects — рабочее пространство для каждой темы

### Что это

**Project** — это контейнер, который хранит одни и те же файлы, инструкции и историю разговоров для одной темы или роли. Один раз настроили — каждый новый чат внутри проекта стартует с этим контекстом.

Аналогия: проект — это рабочая папка. В неё вы один раз положили базу знаний (регламенты, прошлые отчёты, шаблоны), один раз написали инструкции («тон формальный, ссылайся на наш бренд-гайд»). Дальше любой разговор внутри этой папки автоматически работает по этим правилам и с этими файлами.

### Когда нужен проект

| Не нужен (хватит обычного чата) | Нужен |
|---|---|
| Разовый вопрос: «переведи этот email» | Постоянная рутина: еженедельный отчёт |
| Один документ обсудили и забыли | Долгая тема: квартальный план |
| Не нужно помнить контекст между разговорами | Память роли: «я HR-директор и веду команду из 30» |

Типичные проекты руководителя:
- **«Моя команда»** — оргструктура, ИПР, заметки 1-on-1.
- **«Совет директоров»** — устав, прошлые презентации, протоколы.
- **«Бюджет 2026»** — таблицы, прогнозы, переписка.
- **«Корпоративный стиль»** — бренд-гайд, шаблоны писем, позиционирование.
- **«Договоры с подрядчиками»** — рамочный шаблон, история, чек-листы.

### Что внутри проекта

**Project Knowledge** — файлы, которые Claude видит в каждом разговоре проекта:
- Бренд-гайд, корпоративные шаблоны.
- Прошлые отчёты, чтобы новый писался в том же стиле.
- Регламенты, политики, процедуры.
- База знаний продукта.
- Биографии и роли членов команды.

На платных тарифах работает RAG-режим — если знаний много, Claude ищет в них на лету (расширяет «эффективную память» в 10 раз).

**Project Instructions** — постоянные правила для проекта. Пример инструкций для «Совет директоров»:

```
Я операционный директор компании X.
Готовлю материалы для совета директоров (10 человек).

Стиль:
- Деловой, без воды, без эмоций.
- Цифры и факты, минимум прилагательных.
- Заголовки конкретные («Q1 2026: revenue +12%»), не абстрактные.

Структура любого документа:
1. Главное (1 абзац).
2. Цифры (таблицей).
3. Что меняем (3 пункта).
4. Что нужно от совета (1 предложение).

Никогда не выдумывай цифры. Если не знаешь — пиши [НУЖНА ПРОВЕРКА].
```

Этот текст загружается в каждый чат внутри проекта.

**Project Memory** — у каждого проекта своя память, отдельно от обычных чатов и от других проектов. То, что вы рассказали Claude в проекте «Совет директоров», не утечёт в проект «1-on-1 с командой».

### Лимиты

- **Free**: до 5 проектов.
- **Pro и выше**: безлимит.
- **Team / Enterprise**: можно делиться проектом с коллегами (роли «Can use» / «Can edit»).

### Командные проекты

На Team-тарифе один проект можно расшарить — например, «Корпоративный стиль» для всей команды. Кто-то редактирует знания и инструкции, остальные используют. Удобно: один человек ведёт базу, все получают одинаковый Claude.

**Упражнение 4.** Создайте свой первый проект. Тема — то, что вы делаете еженедельно (например, «Еженедельный отчёт» или «1-on-1 с командой»). Загрузите 3–5 файлов, которые точно нужны (шаблон, прошлые версии, инструкция). Напишите Project Instructions: роль, стиль, структура, ограничения. Запустите внутри проекта новый чат — задача та же, что вы делаете каждую неделю. Сравните с обычным чатом без проекта.

## Шаг 5. Memory — Claude помнит между чатами

### Что это

С марта 2026 у Claude есть постоянная память для всех тарифов, включая Free. Раз в сутки Claude автоматически вытаскивает из ваших разговоров устойчивые факты: вашу роль, индустрию, имена коллег, регулярные форматы, личные предпочтения. Эти заметки попадают в Memory и подключаются к каждому новому разговору.

Memory — это не «история чатов». История — это полные тексты прошлых разговоров (можно искать через Settings → Search). Memory — это краткая сводка фактов про вас, которая используется как контекст по умолчанию.

### Что Claude запоминает сам

- «Денис — операционный директор инфобизнеса в России».
- «Команда 12 человек, основные роли — продакт, маркетинг, продажи».
- «Любит короткие ответы без воды».
- «Уже работает с Notion и Slack, не пользуется Asana».

Не запоминает: разовые задачи, чувствительные данные (Anthropic фильтрует), всё, что вы сказали с пометкой «забудь это после ответа».

### Как управлять памятью

**Сказать «запомни X».** Прямо в чате: «Запомни, что наш бренд-голос — деловой, но с лёгкой иронией». Запоминает сразу, не ждёт суточной сводки.

**Сказать «забудь X».** «Забудь, что я работал на компанию Y» — удалит из Memory.

**Просмотр и редактирование.** Settings → Memory. Видите весь список сохранённых заметок, можно удалить точечно.

**Пауза.** Toggle в настройках: Claude временно не использует и не пополняет память. Не удаляет — пауза.

**Сброс.** Кнопка Reset memory — стирает всё. Необратимо.

**Memory в Project отдельная.** Память проекта живёт внутри проекта, не смешивается с общей. Это важно: то, что вы рассказали в проекте «Команда», не утечёт в проект «Личное планирование».

### Когда что использовать: CLAUDE-инструкции, Memory, Project

| Что | Куда положить | Пример |
|---|---|---|
| Постоянное правило для всех чатов | Memory (через «запомни») | «Я предпочитаю короткие ответы без вступлений» |
| Правило для конкретного проекта | Project Instructions | «В отчётах для совета — деловой стиль» |
| Файлы, нужные регулярно в одной теме | Project Knowledge | Бренд-гайд, шаблоны |
| Разовый контекст | Прямо в сообщении | «Сегодня готовлюсь к встрече с подрядчиком» |

### Чему доверять

Memory не идеальна. Иногда Claude запоминает что-то не то или забывает важное. Раз в месяц зайдите в Settings → Memory и проверьте: правда ли там написано про вас. Удалите устаревшее (сменили роль, ушли из компании, сменили стек).

**Упражнение 5.** Откройте Settings → Memory. Прочитайте, что Claude уже про вас знает. Удалите неактуальное. Добавьте через диалог 5 ключевых фактов о вас (роль, команда, индустрия, рабочие инструменты, стиль общения, который вы предпочитаете). Через день начните новый чат и проверьте, использует ли Claude эти факты автоматически.

## Шаг 6. Connectors — подключаем рабочие инструменты

### Что такое connectors

**Connector** — это нативная интеграция Claude с внешним сервисом. Вы один раз даёте Claude доступ через OAuth (как при логине через Google), и дальше можете в чате просить: «посмотри мой календарь», «найди в почте письма от N», «создай страницу в Notion».

Это не то же самое, что MCP (которое есть в курсе для разработчиков). Connectors — это простая кнопка «подключить» в Settings, без YAML и кода.

### Что подключается

На 2026 год доступно 50+ коннекторов, добавляются новые. Самые полезные руководителю:

**Google Workspace** (для всех тарифов):

## Шаг 7. Skills — встроенные навыки и свои команды

### Что такое skill

Skill — это небольшая инструкция или процедура, которую Claude применяет автоматически в подходящей ситуации. Не нужно её вызывать явно — Claude сам понимает по запросу, какой скилл подключить.

Включаются в Settings → Capabilities → Skills.

### Встроенные скиллы Anthropic

Доступны всем, включая Free:

- **xlsx** — продвинутая работа с Excel: формулы, форматирование, многотабовые файлы, графики.
- **docx** — документы Word с реальным форматированием (стили, оглавления, нумерация).
- **pptx** — презентации с дизайнерскими макетами, не просто текст на слайдах.
- **pdf** — генерация PDF из любых документов с правильной вёрсткой.

Когда вы пишете «сделай Excel с финансовой моделью» — автоматически подключается xlsx-скилл, и результат получается в реальном .xlsx-файле, а не в виде «таблицы в чате».

Чтобы скиллы работали, в Settings включён Code Execution (на Pro+ по умолчанию).

### Сторонние и общие скиллы

На Team и Enterprise можно делиться скиллами с коллегами: один человек настраивает «отчёт по нашему шаблону», остальные пользуются. Скиллы появляются в общем каталоге организации.

Есть открытые репозитории сообществом сделанных скиллов — например, `github.com/anthropics/skills`. Можно скачать, упаковать в zip и загрузить в Settings → Skills.

### Свой скилл без программирования

Anthropic выпустил skill-creator — это сам по себе скилл, который интерактивно создаёт другие скиллы через диалог. Вы говорите, что нужно автоматизировать, он задаёт уточняющие вопросы, генерирует структуру и формат. Не нужно писать код.

Пример: «Сделай скилл „еженедельный отчёт“ — запрашивает данные за неделю, форматирует по нашему шаблону, добавляет графики». Skill-creator проведёт через шаги, вы получите готовый файл, кладёте в Settings → Skills → загрузить.

После этого, когда вы или коллега просите «сделай еженедельный отчёт», скилл срабатывает автоматически.

### Когда что использовать

| Что | Для чего |
|---|---|
| Project Instructions | Постоянные правила в одной теме |
| Memory | Постоянные правила про вас в любой теме |
| Skill | Конкретная процедура (создать отчёт, обработать таблицу), повторяемая |
| Plugin | Набор скиллов + коннекторов под целую роль |

**Упражнение 7.** Зайдите в Settings → Capabilities → Skills, посмотрите, что включено. Проверьте на одной задаче эффект встроенных скиллов: попросите сделать финансовую модель в Excel или презентацию из черновика — увидите разницу с обычной выгрузкой текста. Если есть повторяющаяся задача (отчёт, рассылка, протокол) — попробуйте через skill-creator превратить её в скилл.

## Шаг 8. Cowork — Claude как агент-исполнитель

### Что это

**Cowork** — это режим Claude Desktop, в котором он становится не собеседником, а исполнителем. Вы даёте задачу — он сам открывает приложения, работает с файлами на компьютере, пишет, считает, конвертирует, организует. Не подсказывает, как сделать, а делает.

Это самое значимое отличие от обычного чата:
- **Чат** — Claude советует. Файлы вы загружаете руками, результат скачиваете руками.
- **Cowork** — Claude делает. Получает доступ к папке, работает с файлами напрямую, оставляет результат в той же папке.

Cowork работает в изолированной виртуальной машине внутри Desktop-приложения — ваш компьютер защищён, но в выбранной папке Claude может реально менять файлы.

### Тарифы и установка

- Только **Pro и выше**.
- Только **Claude Desktop** (Mac, Windows). На веб-версии и мобильном — нет.
- В Desktop-приложении переключаетесь между режимами: Chat / Cowork / Code.

### Как давать задачу

1. В Desktop-приложении нажимаете на вкладку Cowork.
2. Внизу — чекбокс «Work in a Folder».
3. Выбираете папку (Downloads, Documents, рабочая папка).
4. Подтверждаете permission: разовый доступ или постоянный.
5. Пишете задачу обычным языком.

Cowork составляет план, спрашивает подтверждения по ключевым шагам, выполняет.

### Реальные сценарии для руководителя

**1. Разобрать Downloads.**
> Папка Downloads — 200+ файлов: чеки, скрины, документы, дубли. Разбери: создай подпапки по типам, переименуй файлы по содержимому (вместо «IMG_2839.png» — описательно), удали дубли.

Реальный замер из туториалов: 186 файлов → 11 подпапок, 27 дублей удалено по криптографическим хэшам (а не по именам), все файлы переименованы по содержимому.

**2. Конвертация пачки документов.**
> Все .docx из этой папки — в PDF. Все .png — оптимизировать. Все .pdf — сжать.

Cowork использует установленные локально инструменты (LibreOffice, Ghostscript) — без облачных конвертеров.

**3. Аналитический отчёт из выгрузки.**
> Возьми CSV-выгрузку из CRM, посчитай выручку по сегментам, нарисуй графики, оформи 10-страничный PDF-отчёт с выводами.

Cowork сам разберёт CSV, посчитает, сделает визуализации, сверстает PDF.

**4. Регулярная задача по расписанию.**
Cowork умеет recurring tasks — задаёте «каждое утро в 8:00 собери дайджест из Slack, Gmail и календаря, положи в Notion».

**5. Подготовка к совещанию.**
> Возьми протоколы из папки «Совещания», вытащи незакрытые action items со ссылками на ответственных, оформи Excel-чеклист с дедлайнами.

### Permission-режимы

- **Ask before each step** — Claude спрашивает перед каждым серьёзным действием. Безопаснее, медленнее.
- **Approve plan, then execute** — вы утверждаете общий план, дальше Claude действует сам в рамках плана. Быстрее, нужно доверять.
- **Прямой запрет на удаление** — Cowork всегда спрашивает перед физическим удалением файлов.

### Ограничения, которые важно знать

- Сессия живёт, пока приложение открыто. Закрыли — задача прервалась.
- Cowork жрёт больше токенов, чем чат — задача из 30 шагов = десятки сообщений модели.
- **Аудит и compliance: активность Cowork НЕ попадает в Audit Logs и Compliance API.** Не используйте Cowork для регулируемых данных (HIPAA, GDPR-чувствительное, банковское).
- Память внутри Cowork-сессии — внутри Project. Между обычными Cowork-сессиями (вне проектов) память не сохраняется.
- Гугл-коннекторы в Cowork ещё дорабатываются (на момент 2026 — лучше работают через обычный чат).

### Cowork vs Chat: когда что

| Задача | Чат | Cowork |
|---|---|---|
| Один документ, один вопрос | ✅ | излишне |
| Разобрать папку | ❌ | ✅ |
| Сделать отчёт из выгрузки | ✅ (если выгрузка одна) | ✅ (если данные в нескольких файлах) |
| Регулярная еженедельная задача | ❌ (надо запускать вручную) | ✅ (по расписанию) |
| Multi-step с сохранением промежуточных файлов | ❌ | ✅ |
| Регулируемые данные | ✅ (Team/Enterprise) | ❌ |

**Упражнение 8.** Включите Cowork в Desktop-приложении. Дайте ему задачу «разобрать Downloads / Документы»: создать подпапки по тематике, переименовать файлы, удалить дубли. Контролируйте permission на «Ask before each step» в первый раз — посмотрите, как он рассуждает. Потом переключите на «Approve plan» и дайте задачу посерьёзнее: подготовить отчёт из нескольких файлов, конвертировать пачку документов, настроить регулярную задачу.

## Шаг 9. Plugins — готовые наборы под роли

**Plugin** — это упакованный набор: скиллы + коннекторы + slash-команды + субагенты, заточенные под конкретную роль. Anthropic выпустил 11 официальных плагинов, открытый репозиторий `github.com/anthropics/knowledge-work-plugins`. Доступны на Pro+, ставятся через Cowork.

Аналогия: если skill — отдельный приём, то plugin — школа боевых искусств целиком. Ставите плагин «Sales» — Claude получает понимание пайплайна, подключения к HubSpot и Slack, готовые команды `/sales:call-prep` и формат подготовки к звонку.

- **Productivity** — обязателен почти всем. Связывает почту, календарь, мессенджеры, таск-менеджеры.

- **Sales / Finance / Marketing / PM** — по профилю.
- **Plugin Create** — когда поняли, что хочется свой набор под именно вашу роль.

### Установка

1. Открываете Cowork в Claude Desktop.
2. Settings → Plugins.
3. Выбираете плагин, нажимаете Install.
4. Авторизуете нужные коннекторы (OAuth).

2. Запустить, описать роль и задачи.

Минимально нужно знать: что именно автоматизировать. Технических навыков — не нужно.

## Шаг 10. Claude в Chrome — агент в браузере

### Что это

Расширение для Chrome (и Chromium-based браузеров). Открываете боковую панель — Claude видит ту же страницу, что и вы. Может читать, кликать, заполнять формы, переходить по ссылкам. Полностью автономно или с подтверждением каждого шага.

### Когда полезно руководителю

- **Заполнение форм.** Экспенс-репорты, тревел-заявки, табели — все эти корпоративные формы, которые вы ненавидите.
- **Регистрации.** На конференции, вебинары, в сервисы.
- **Сбор данных с сайтов без API.** Цены конкурентов, открытые вакансии, новости индустрии.
- **Multi-tab задачи.** Открыто 5 вкладок про одну тему — Claude свёл в один документ.
- **Заказы и бронирования.** Переговорки, билеты, такси (с подтверждением, естественно).
- **Отчёты по дашбордам, у которых нет export.** Скрины + чтение → нормальный отчёт.

### Permission-режимы

- **Ask before acting** (по умолчанию). Claude перед каждым действием показывает, что собирается сделать. Вы соглашаетесь или меняете план.
- **Approve plan, then act.** Согласовали общий план — дальше Claude работает сам. На критических действиях (платёж, отправка письма, передача данных) останавливается.
- **Act without asking.** Полный автопилот для рутины, которой вы доверяете. Включается осознанно.

### Multi-tab

Перетаскиваете нужные вкладки в специальную «группу Claude». Он видит все одновременно, может между ними переключаться, сводить данные. Полезно, когда задача — «возьми из 5 разных мест и сделай одно».

### Тарифы и модели

- **Pro** — расширение работает, но только на модели Haiku (быстрая, попроще).
- **Max / Team / Enterprise** — выбор моделей до Opus (сложные многошаговые задачи).

### Безопасность

- Не оставляйте «Act without asking» включённым на сайтах с платежами или корпоративных системах.
- Не давайте доступ к админкам, в которых одно неправильное действие — катастрофа.
- Чувствительные пароли и 2FA — Claude видит, что на экране. Стоит понимать.
- Расширение можно ставить на паузу или отключать на конкретных сайтах.

**Упражнение 10.** Поставьте расширение. Выберите рутинную, но безопасную задачу: заполнить форму отзыва, оставить комментарий на странице, скачать пару отчётов с дашборда. Поработайте в режиме «Ask before acting», чтобы понять логику. Попробуйте multi-tab: откройте 3 страницы с похожей информацией, попросите Claude свести в один документ.

## Сквозные сценарии руководителя

Это шаблоны, как комбинировать всё освоенное. Каждый сценарий — реальная рутина, которую можно автоматизировать на 60–90%.

### Сценарий 1. Утро менеджера (15 минут вместо 2 часов)

Что делается обычно: открыли почту, разобрали 80 писем, открыли Slack, прочитали 5 каналов, открыли календарь, посмотрели сетку, прикинули приоритеты.

Что делает Claude (Pro + Productivity-плагин):
1. Утром вы пишете в Project «Утренняя планёрка»: «дайджест».
2. Claude через connectors собирает: непрочитанная почта (с приоритетами и срочными), вчерашние Slack-обсуждения по вашим каналам (саммари), сетка на сегодня (с предупреждениями про конфликты).
3. Артефакт-документ: «Утро 29.04», 200–300 слов, по разделам.
4. Опционально: автоматический скилл создаёт задачи в Notion из писем с дедлайнами.

Что нужно: Pro, Project, коннекторы Gmail / Slack / Calendar, плагин Productivity. Можно вызывать вручную, можно настроить через Cowork как recurring task на 8:00.

### Сценарий 2. Совещание под ключ

**До совещания.** Project «Командные планёрки» с регламентом и прошлыми протоколами:
> Завтра планёрка с командой. Подготовь повестку: возьми незакрытые action items из последних 3 протоколов, добавь новые темы из чата с командой за неделю, поставь тайминг.

**Во время.** Включаете запись (Otter, Fireflies) — расшифровка появится автоматически. Или диктуете тезисы голосом в Claude на телефоне.

**После.** Загружаете транскрипт:
> Сделай протокол: главное, решения, action items с ответственными и дедлайнами, открытые вопросы.

Артефакт-документ. Через коннекторы — задачи переносятся в Asana, рассылка протокола в Slack.

### Сценарий 3. Отчёт для босса

Project «Отчёты руководству» с шаблоном, прошлыми отчётами, фирменным стилем.

В Project Knowledge: бренд-гайд, прошлые 3 квартальных отчёта, KPI-метрики, оргструктура.

В Project Instructions:
- Стиль: цифры в начале, контекст после, выводы и предложения в конце.
- Все цифры — со ссылкой на источник или [НУЖНА ПРОВЕРКА].
- Не больше 2 страниц.

Регулярный запрос:
> Сделай еженедельный отчёт по моей команде. Данные — в прикреплённых выгрузках.

Артефакт PowerPoint или Word, готовый к рассылке.

### Сценарий 4. Документы и регламенты

Project «Документы компании» с действующими регламентами и шаблонами.

Типичные задачи:
- «Сделай служебку на закупку оборудования. Контекст в файле».
- «Проверь этот черновик регламента на соответствие нашему бренд-гайду и юридической логике».
- «Перепиши этот документ из формального стиля в человеческий, для рассылки команде».

Cowork — для пакетных задач: «Возьми все служебки в этой папке, переформатируй по новому шаблону, сохрани с тегами в названии».

### Сценарий 5. Аналитика для нетехнаря

У вас Excel-выгрузка из ERP, 12 табов, тысячи строк. Раньше — звонили аналитику.

Claude (с включёнными xlsx-скиллами):
1. Загружаете файл.
2. «Найди 5 главных аномалий за апрель vs март. Объясни возможные причины. Дай рекомендации».
3. Артефакт-дашборд (HTML с интерактивными графиками) или Excel со сводными таблицами.
4. Через Drive-коннектор — кладёте в общую папку команде.

Лимиты: сложные многотабовые экселки с merged cells и неструктурированными зонами Claude разбирает хуже. Помогает: приведите файл к одной плоской таблице с заголовками — точность взлетает.

### Сценарий 6. Команда и 1-on-1

Project «Команда» с заметками о каждом сотруднике (имя, роль, ИПР, история встреч). Memory держит общие принципы вашего стиля управления.

Регулярные задачи:
- «Подготовь повестку для 1-on-1 с N. Возьми прошлые встречи и его последние коммиты / задачи в Asana».
- «N просит обратную связь по этому проекту. Помоги мне сформулировать структурированно: что хорошо, что улучшить, что конкретно сделать дальше».
- «У меня сложный разговор с M. Помоги отрепетировать: дай 3 возможных реакции и как на них отвечать».

### Сценарий 7. Браузерная рутина

Claude в Chrome. Раз в неделю — экспенс-репорты, бронирования, формы.

> Вот скрины чеков за неделю. Заполни форму экспенс-репорта в нашей системе. Категории — по моим прошлым отчётам.

> Забронируй переговорку на 14:00 завтра, пригласи N и M. Тема — «Q2 review».

Не используйте на финансовых платформах с правами на перевод денег и в админках, где одно действие необратимо.

## Безопасность и риски

Главный принцип: **что Claude видит — то Anthropic потенциально видит**. Технически данные передаются через серверы Anthropic.

| Тариф | Обучение на ваших данных | Аудит-логи | HIPAA / SOC2 |
|---|---|---|---|
| Free | По умолчанию **может** (выкл. в настройках) | Нет | Нет |
| Pro | По умолчанию **может** (выкл. в настройках) | Нет | Нет |
| Max | Так же, как Pro | Нет | Нет |
| Team | По умолчанию **не использует** | Базовые | Нет |
| Enterprise | По умолчанию **не использует** | Полные, Compliance API, Data Export | HIPAA-ready |

**Что НЕ давать Claude никогда:**
- Пароли, ключи, токены доступа.
- Банковские реквизиты, номера карт.
- Медицинская информация (HIPAA — только Enterprise).
- Персональные данные сотрудников / клиентов в значимом объёме (GDPR — Team минимум, лучше Enterprise).
- Государственная или коммерческая тайна.

**Cowork отдельно**: его активность не попадает в Audit Logs и Compliance API даже на Enterprise. Для регулируемой работы Cowork не использовать.

**Connectors отдельно**: OAuth даёт Claude доступ к вашим данным в Gmail / Drive / Slack. Если в компании есть DLP-система или регламент на сторонние интеграции — согласуйте до подключения.

**Что делать в первую неделю:**
1. Settings → Privacy → Help improve Claude → выключить (если не на Team/Enterprise).
2. Settings → Memory → проверить, что там не записано лишнего.
3. Settings → Connectors → проверить, какие сервисы подключены, отозвать ненужные.
4. На стороне Google / Microsoft / Notion — проверить список приложений с доступом.

## Когда нужен Claude Code (бонус)

В этом же репозитории есть [методичка по Claude Code](README.md) — продвинутая версия для тех, кто работает в терминале. Зачем она нужна руководителю:

- Если в команде есть разработчики и вы хотите понимать, чем они пользуются.
- Если вы дорастаете до автоматизаций, которые требуют скриптов и кода.
- Если хотите подключать MCP-серверы (более глубокая интеграция, чем коннекторы).
- Если строите AI-пайплайны для регулярного производства контента, отчётов, аналитики.

Для большинства руководителей хватает Claude.ai + Cowork + плагинов. Claude Code — следующий уровень, когда упёрлись в потолок простых инструментов.

## Чек-лист на первую неделю

**День 1.** Регистрация. Pro-тариф. Десктоп + мобильное приложение. Обход настроек Privacy и Memory.

**День 2.** Первый рабочий проект (Project + Knowledge + Instructions) на любую вашу регулярную задачу. Один реальный артефакт-документ внутри проекта.

**День 3.** Подключить Gmail, Calendar, Notion / Asana. Сделать утренний дайджест.

**День 4.** Memory — добавить 5 фактов о себе и команде через диалог. Проверить через сутки, что Claude помнит.

**День 5.** Установить Cowork. Дать задачу «разобрать Downloads» в режиме «Ask before each step».

**Выходные.** Подумать, какие 3 рутинные задачи у вас занимают больше 30 минут в неделю каждая. На следующей неделе — собирать под них Project + плагин + при необходимости свой скилл через skill-creator.

**Через месяц** — половина рутины должна закрываться автоматически. Если нет — пересмотреть, где конкретно вы недоинвестировали в настройку.
