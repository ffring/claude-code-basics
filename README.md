# Claude Code: методология работы с агентом

Последовательная методичка по работе с Claude Code как с инженерным агентом: от установки до полного автоматизированного пайплайна. Восемь шагов, каждый строится на предыдущем. Сквозной кейс — автоматический AI-дайджест: блог, который сам ищет новости, пишет обзоры, генерирует обложки, коммитит в Git и публикует выпуски.

Методология даёт ответ на главный вопрос: как сделать так, чтобы агент стабильно выдавал проверяемый результат, а не превращал работу в бесконечный цикл правок.

## Принцип

Работа с агентом — это не «попросил — получил». Это конструирование среды, в которой агент действует точно: правильный контекст, точные инструменты, понятные правила, автоматические проверки. Эта среда называется **Agent Harness** — обвязка вокруг модели. Курс собирает её слой за слоем.

| Шаг | Слой | Что добавляется в harness |
|-----|------|---------------------------|
| 1 | Окружение | Установка, права доступа, базовый `CLAUDE.md` |
| 2 | Промптинг | Формат задачи: цель, контекст, ограничения, проверка |
| 3 | Планирование | Plan Mode — сначала план, потом правки |
| 4 | Контекст | Память, правила, постепенное раскрытие знаний |
| 5 | MCP | Подключение внешних сервисов и источников данных |
| 6 | Навыки и хуки | Повторяющиеся действия и автоматические проверки |
| 7 | Оркестрация | Несколько агентов параллельно, изоляция через worktrees |
| 8 | Harness | Полный цикл, отладка, headless, SDK, CI/CD |

Папки `01-base`...`08-harness` идут в той же последовательности и содержат презентации и PDF-упражнения для каждого шага.

## Шаг 1. Основы

📂 [`01-base`](01-base/) · [презентация](01-base/automatica-ccb-s1-e1.pdf) · упражнения: [01](01-base/exercise/01-setup-and-verify.pdf), [02](01-base/exercise/02-claude-md-creation.pdf), [03](01-base/exercise/03-permission-modes.pdf), [04](01-base/exercise/04-basic-commands.pdf), [05](01-base/exercise/05-digest-blog-init.pdf)

Цель — поднять рабочее окружение и понять, чем агент управляется. Установка Claude Code, проверка через `claude doctor`, разбор папки `.claude/`, первый `CLAUDE.md` (глобальный и проектный), режимы доступа.

Что осваиваем:
- Установка и `claude doctor`.
- Структура `.claude/`: настройки, правила, навыки, агенты.
- Иерархия `CLAUDE.md`: managed → global → project → local.
- Что пишем в `CLAUDE.md`, а что отдаём линтерам и тестам.
- Режимы доступа: Default, Accept Edits, Plan, Don't Ask, Bypass Permissions.
- Команды: `/clear`, `/compact`, `/memory`, `/cost`, `/model`, `/config`, `/permissions`.
- Возобновление сессий: `--continue`, `--resume`, именованные диалоги.

**Результат шага**: окружение настроено, `CLAUDE.md` написан, инициализирован репозиторий дайджеста.

## Шаг 2. Промптинг

📂 [`02-prompting`](02-prompting/) · [презентация](02-prompting/automatica-ccb-s1-e2.pdf) · упражнения: [01](02-prompting/exercise/01-structured-vs-vague.pdf), [02](02-prompting/exercise/02-git-workflow.pdf), [03](02-prompting/exercise/03-anti-patterns.pdf), [04](02-prompting/exercise/04-digest-prompts.pdf)

Задача для агента — не запрос к чат-боту. Агенту нужны цель, контекст, ссылки на файлы, ограничения и способ проверить результат. Параллельно осваиваем Git как страховку: ветки, маленькие коммиты, откаты.

Что осваиваем:
- Структура агентного промпта: контекст → файлы → задача → формат → ограничения.
- Постановка через цель, а не через команду «сделай».
- Три уровня конкретности: исследование, направление, точная спецификация.
- Ограничения, не дающие агенту менять лишнее.
- Промпты с критериями проверки и командами тестов.
- Управление глубиной рассуждений модели.
- Git-workflow вокруг агента.
- Антипаттерны: перегруженный промпт, разговор-свалка, бесконечный цикл правок.

**Результат шага**: набор промптов для дайджеста и рабочий процесс с ветками и откатами.

## Шаг 3. Планирование

📂 [`03-planning`](03-planning/) · [презентация](03-planning/automatica-ccb-s1-e3.pdf) · упражнения: [01](03-planning/exercise/01-plan-mode.pdf), [02](03-planning/exercise/02-ide-integration.pdf), [03](03-planning/exercise/03-remote-control.pdf), [04](03-planning/exercise/04-digest-architecture.pdf)

Когда задача требует анализа, агенту нельзя сразу разрешать править код. Сначала он исследует проект, собирает факты и предлагает план. Только после одобрения — изменения.

Что осваиваем:
- Как Claude Code выбирает инструменты, действует и читает результат.
- Plan Mode: режим исследования без правки файлов.
- Когда нужен план: незнакомая кодовая база, крупный рефакторинг, архитектурные решения.
- Передача контекста через `@`-упоминания файлов, папок, строк.
- Контрольные точки и откаты.
- Интеграция с JetBrains: компилятор, diff в IDE.
- Удалённое управление активной сессией.

**Результат шага**: архитектура дайджеста собрана через цикл `Plan Mode → план → правки → коммит`.

## Шаг 4. Контекст

📂 [`04-context`](04-context/) · [презентация](04-context/automatica-ccb-s1-e4.pdf) · упражнения: [01](04-context/exercise/01-context-management.pdf), [02](04-context/exercise/02-memory-setup.pdf), [03](04-context/exercise/03-advanced-claude-md.pdf), [04](04-context/exercise/04-digest-context-workflow.pdf) · видео: [LLMs explained briefly](04-context/Large%20Language%20Models%20explained%20briefly_@aiwizards_720.mp4)

Контекстное окно — рабочая память агента. Оно забивается историей, логами, файлами, выходом инструментов. Раздутый контекст снижает точность модели. Этот шаг — про то, как держать контекст в форме.

Что осваиваем:
- Из чего состоит контекст: системный промпт, инструменты, агенты, память, история, свободное место.
- Почему раздутый контекст бьёт по точности.
- Что засоряет: длинный `CLAUDE.md`, лишние MCP-серверы, неочищенная история.
- Сжатие: автоматическое и ручное через `/compact`.
- Команды: `/context`, `/clear`, `/rewind`, `/model`.
- `/memory`, auto-memory, файл `MEMORY.md`.
- Постепенное раскрытие: короткий `CLAUDE.md`, тематические правила в `.claude/rules/`, ссылки через `@path`.

**Результат шага**: правила, память и рабочие файлы разнесены по местам, агент работает без перегрузки контекста.

## Шаг 5. MCP и внешние системы

📂 [`05-mcp`](05-mcp/) · [презентация](05-mcp/automatica-ccb-s1-e5.pdf) · упражнения: [01](05-mcp/exercise/01-mcp-architecture.pdf), [02](05-mcp/exercise/02-tavily-search.pdf), [03](05-mcp/exercise/03-digest-first-publish.pdf), [04](05-mcp/exercise/04-group-mcp-challenge.pdf)

Model Context Protocol даёт агенту доступ к внешним сервисам: GitHub, поиск, базы данных, Figma, Slack. Здесь — как устроены MCP-серверы, как их подключать, где границы безопасности.

Что осваиваем:
- Архитектура MCP: хост, клиент, сервер.
- Что MCP отдаёт агенту: инструменты, ресурсы, промпты.
- Подключения: HTTP, SSE, stdio.
- `claude mcp add` и `.mcp.json`.
- Области видимости: local, project, user.
- Хранение ключей в переменных окружения.
- Поиск серверов: PulseMCP, mcp.so, MCP Registry.
- MCP Inspector для отладки.

**Результат шага**: дайджест ищет новости через Tavily, готовит графику через Replicate, работает с репозиторием через GitHub MCP.

## Шаг 6. Навыки и автоматизация

📂 [`06-skills-hooks`](06-skills-hooks/) · [презентация](06-skills-hooks/automatica-ccb-s1-e6.pdf) · упражнения: [01](06-skills-hooks/exercise/01-custom-skills.pdf), [02](06-skills-hooks/exercise/02-hooks-setup.pdf), [03](06-skills-hooks/exercise/03-tdd-workflow.pdf), [04](06-skills-hooks/exercise/04-digest-skill.pdf)

Повторяющиеся команды превращаем в навыки и хуки. **Навык** описывает как выполнять задачу. **Хук** срабатывает в нужный момент работы агента — до или после действия.

Что осваиваем:
- Пользовательские навыки в `.claude/skills/`.
- YAML-шапка навыка: инструменты, модель, изоляция контекста, хуки.
- Аргументы через `$ARGUMENTS`, повторный запуск через `/loop`.
- События хуков: PreToolUse, PostToolUse, Stop и другие.
- Типы хуков: shell-команды, HTTP-запросы, LLM-проверки.
- Применение: блокировка опасных действий, автоформатирование, запуск тестов.
- TDD-цикл `RED → GREEN → REFACTOR`.

**Результат шага**: навыки для выпуска дайджеста и генерации обложек, хук-проверка качества статьи.

## Шаг 7. Агенты и оркестрация

📂 [`07-agents`](07-agents/) · [презентация](07-agents/automatica-ccb-s1-e7.pdf) · упражнения: [01](07-agents/exercise/01-sub-agents.pdf), [02](07-agents/exercise/02-agent-teams.pdf), [03](07-agents/exercise/03-worktrees.pdf), [04](07-agents/exercise/04-digest-parallel-agents.pdf)

Один агент превращается в команду: один исследует, второй пишет, третий готовит обложки, четвёртый собирает страницы. У каждого — своя задача, свой контекст, при необходимости — отдельная рабочая копия кода.

Что осваиваем:
- Встроенные субагенты: Explore, Plan, General-purpose.
- Собственные агенты в `.claude/agents/`.
- Отдельное контекстное окно для каждого субагента.
- Фоновые задачи и команда `/tasks`.
- Agent Teams: общий список задач и обмен сообщениями.
- Статусы задач: pending, in_progress, completed.
- `claude --worktree` для изоляции изменений в параллельных копиях.

**Результат шага**: несколько агентов параллельно ищут, пишут, рисуют и собирают дайджест.

## Шаг 8. Agent Harness

📂 [`08-harness`](08-harness/) · [презентация](08-harness/automatica-ccb-s1-e8.pdf) · упражнения: [01](08-harness/exercise/01-debugging.pdf), [02](08-harness/exercise/02-headless-mode.pdf), [03](08-harness/exercise/03-sdk-intro.pdf), [04](08-harness/exercise/04-digest-full-pipeline.pdf)

Финальный шаг — собрать всё в работающий цикл. Harness — это правила, инструменты, доступы, навыки, хуки, агенты и проверки в одном пайплайне. Параллельно — отладка, когда агент застревает или чинит одно, ломая другое.

Что осваиваем:
- Из чего состоит Agent Harness.
- Минимизация инструментов: агенту даём только нужное.
- Линтеры и тесты как автоматическая проверка правок.
- Визуальная проверка: скриншоты, Playwright MCP.
- Проверка результата другой моделью (LLM-as-judge).
- Отладка: логи, стек-трейсы, воспроизводимые ошибки.
- Headless-режим для скриптов и CI/CD.
- GitHub Actions: issues, pull requests, комментарии.
- SDK: `query()`, `ClaudeSDKClient`, собственные инструменты.

**Результат шага**: полный цикл AI-дайджеста — поиск тем → текст → графика → проверка → коммит → публикация. Запускается автоматически.

## Сквозной проект: AI-дайджест

Проект растёт по шагам:

1. Репозиторий и базовые редакционные правила.
2. Промпты для поиска, отбора и подготовки статей.
3. Архитектура сайта и порядок публикации.
4. Правила и знания вынесены из контекста в память и отдельные файлы.
5. Подключены Tavily (поиск), Replicate (графика), GitHub (репозиторий).
6. Повторяющиеся действия оформлены как навыки и хуки.
7. Работа разделена между агентами: поиск, текст, обложки, страницы.
8. Весь цикл запускается автоматически.

## Что появится в навыках после прохождения

- Настройка Claude Code, прав доступа и рабочих команд.
- Написание `CLAUDE.md`, который помогает агенту, а не дублирует линтеры.
- Постановка задач через цель, контекст, ограничения, критерии проверки.
- Использование Git как страховки.
- Plan Mode для незнакомого кода и архитектурных решений.
- Чистка контекста, вынос правил в память и подключаемые файлы.
- Подключение внешних систем через MCP.
- Создание навыков, хуков и автоматических проверок.
- Запуск субагентов и работа с worktrees без конфликтов.
- Headless-режим, GitHub Actions и SDK.
- Сборка собственного Agent Harness.

## Как проходить

1. Идти по папкам последовательно: `01-base` → `08-harness`.
2. В каждой папке сначала открыть презентацию.
3. Затем выполнить PDF-упражнения из `exercise/` по порядку.
4. После каждого шага — фиксировать изменения в Git.
5. Не складывать всё в один `CLAUDE.md`: по мере курса переносить правила в память, отдельные файлы, навыки и хуки.

Пассивно методичку не пройти — нужно выполнять упражнения и собирать проект руками.

## Требования

- macOS, Windows или Linux.
- VS Code или другая IDE.
- Git и базовые навыки терминала.
- Доступ к Claude Code.
- Для практики — API-ключи Tavily, Replicate, GitHub, Vercel.

## Кому подойдёт

- Разработчикам, которые хотят сделать Claude Code основным инструментом.
- Техническим специалистам, автоматизирующим рутину.
- Пользователям Claude Code, Cursor и аналогов, которым нужен проверяемый результат.
- Тем, кто хочет разобраться в MCP, хуках, навыках, субагентах и агентных пайплайнах на практике.
