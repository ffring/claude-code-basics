# Основы Claude Code

Открытые материалы курса [Claude Code Basics](https://www.automatica.io/courses/claude-code-basics) от Automatica.

Курс рассказывает, как работать с Claude Code как с агентом для разработки: настраивать окружение, ставить задачи, управлять контекстом, подключать внешние сервисы через MCP, создавать навыки, запускать хуки и распределять работу между несколькими агентами.

## Что в репозитории

- PDF-презентации восьми занятий.
- Практические задания в папках `exercise/`.
- Сквозной проект курса — автоматический AI-дайджест.
- Видео к занятию об управлении контекстом (перевод видео 3blue1brown в четвёртом занятии).

Папки идут в том же порядке, что и занятия:

| № | Папка | Тема | Что получится в конце |
|---|-------|------|-----------------------|
| 1 | [`01-base`](01-base/) | Основы | Установите Claude Code, создадите стартовый проект и первый `CLAUDE.md`. |
| 2 | [`02-prompting`](02-prompting/) | Промптинг | Научитесь ставить агенту задачи так, чтобы результат было легко проверить. |
| 3 | [`03-planning`](03-planning/) | Планирование | Пройдёте этапы планирования через Plan Mode: сначала план, потом правки. |
| 4 | [`04-context`](04-context/) | Контекст | Научитесь работать с контекстом чтобы агент не терял фокус. |
| 5 | [`05-mcp`](05-mcp/) | MCP | Подключите Claude Code к внешним сервисам и источникам данных. |
| 6 | [`06-skills-hooks`](06-skills-hooks/) | Навыки и хуки | Соберёте навыки и проверки для повторяющихся задач. |
| 7 | [`07-agents`](07-agents/) | Агенты | Запустите несколько агентов параллельно, синхронизируете изменения. |
| 8 | [`08-harness`](08-harness/) | Agent Harness | Соберёте обвязку (harness), которая запускает полный цикл автоматизации. |

## Курсовой проект

Во время курса вы разрабатываете автоматический AI-дайджест. Это блог, который ищет новости по заданной теме, отбирает важные, пишет короткие обзоры, генерирует обложки, сохраняет изменения в Git и публикует новые выпуски.

Проект растёт вместе с курсом:

- **Ступень 1**: создаёте репозиторий и базовые редакционные правила.
- **Ступень 2**: пишете промпты для поиска, отбора и подготовки статей.
- **Ступень 3**: проектируете структуру сайта и порядок публикации.
- **Ступень 4**: выносите правила и знания из переполненного контекста в память и отдельные файлы.
- **Ступень 5**: подключаете Tavily для поиска, Replicate для графики и GitHub для работы с репозиторием.
- **Ступень 6**: оформляете повторяющиеся действия как навыки и хуки.
- **Ступень 7**: делите работу между агентами: поиск, текст, обложки и страницы.
- **Ступень 8**: запускаете весь цикл автоматически.

## После курса

Вы сможете:

- настроить Claude Code, права доступа и рабочие команды;
- написать `CLAUDE.md`, который помогает агенту, но не пытается заменить линтеры и тесты;
- ставить задачи через цель, контекст, ограничения и критерии проверки;
- использовать Git как страховку при работе с агентом;
- включать Plan Mode для незнакомого кода, архитектурных решений и рефакторинга;
- чистить и сжимать контекст, выносить правила в память и подключаемые файлы;
- подключать внешние системы через Model Context Protocol (MCP);
- создавать навыки, хуки и автоматические проверки;
- запускать субагентов и работать с `worktrees`, чтобы изменения не конфликтовали;
- использовать Claude Code без интерактивного интерфейса: в `headless`-режиме, GitHub Actions и SDK;
- собирать Agent Harness — набор правил, инструментов и проверок вокруг модели.

## Программа

### 1. Основы

Материалы: [презентация](01-base/automatica-ccb-s1-e1.pdf), упражнения [01](01-base/exercise/01-setup-and-verify.pdf), [02](01-base/exercise/02-claude-md-creation.pdf), [03](01-base/exercise/03-permission-modes.pdf), [04](01-base/exercise/04-basic-commands.pdf), [05](01-base/exercise/05-digest-blog-init.pdf).

Вы устанавливаете Claude Code, проверяете окружение через `claude doctor`, разбираетесь в содержимом папки `.claude/`, создаёте глобальный и проектный `CLAUDE.md`, настраиваете режимы доступа и осваиваете команды, которые нужны в ежедневной работе.

Темы:

- установка и настройка Claude Code;
- папка `.claude/`: настройки, правила, навыки и агенты;
- иерархия `CLAUDE.md`: управляемые, глобальные, проектные и локальные инструкции;
- что агент должен знать о проекте;
- что держать в `CLAUDE.md`, а что отдать линтерам и тестам;
- режимы доступа: Default, Accept Edits, Plan, Don’t Ask, Bypass Permissions;
- команды `/clear`, `/compact`, `/memory`, `/cost`, `/model`, `/config`, `/permissions`;
- возобновление сессий через `--continue`, `--resume` и имена диалогов.

**Результат**: Claude Code готов к работе, правила доступа понятны, в проекте есть первый `CLAUDE.md` и стартовый репозиторий дайджеста.

### 2. Промптинг

Материалы: [презентация](02-prompting/automatica-ccb-s1-e2.pdf), упражнения [01](02-prompting/exercise/01-structured-vs-vague.pdf), [02](02-prompting/exercise/02-git-workflow.pdf), [03](02-prompting/exercise/03-anti-patterns.pdf), [04](02-prompting/exercise/04-digest-prompts.pdf).

Вы узнаете, чем задача для кодового агента отличается от запроса к чат-боту. Агенту нужны цель, контекст, ссылки на файлы, ограничения и способ проверить результат. Отдельный блок посвящён Git: веткам, небольшим коммитам, откату изменений и ревью правок.

Темы:

- структура агентного промпта: контекст, файлы, задача, формат, ограничения;
- почему постановку задачи лучше начинать с цели, а не с команды «сделай»;
- три уровня конкретности: исследование, направление, точная спецификация;
- ограничения, которые не дают агенту менять лишнее;
- промпты с критериями проверки и командами запуска тестов;
- управление глубиной рассуждения модели;
- Git как безопасная рамка для экспериментов с агентом;
- типичные ошибки: перегруженный промпт, разговор-свалка, бесконечный цикл правок.

**Результат**: набор промптов для выпуска дайджеста и рабочий процесс с ветками, коммитами и откатами.

### 3. Планирование

Материалы: [презентация](03-planning/automatica-ccb-s1-e3.pdf), упражнения [01](03-planning/exercise/01-plan-mode.pdf), [02](03-planning/exercise/02-ide-integration.pdf), [03](03-planning/exercise/03-remote-control.pdf), [04](03-planning/exercise/04-digest-architecture.pdf).

Вы учитесь не просить агента сразу править код когда задача требует дополнительного анализа. Сначала агент исследует проект, собирает факты и предлагает план. Только после этого вы разрешаете изменения.

Темы:

- как Claude Code выбирает инструменты, выполняет действия и читает результат;
- Plan Mode: режим исследования без правки файлов;
- когда нужен план: незнакомая кодовая база, крупный рефакторинг, архитектурные решения;
- передача контекста через `@`-упоминания файлов, папок и строк кода;
- контрольные точки для отката разговора или изменений;
- работа с JetBrains: диагностика компилятора и просмотр diff в IDE;
- удалённое управление активной сессией.

**Результат**: архитектура дайджеста оформлена через цикл `Plan Mode -> план -> правки -> коммит`.

### 4. Управление контекстом

Материалы: [презентация](04-context/automatica-ccb-s1-e4.pdf), упражнения [01](04-context/exercise/01-context-management.pdf), [02](04-context/exercise/02-memory-setup.pdf), [03](04-context/exercise/03-advanced-claude-md.pdf), [04](04-context/exercise/04-digest-context-workflow.pdf), видео [LLMs explained briefly](04-context/Large%20Language%20Models%20explained%20briefly_@aiwizards_720.mp4).

Контекстное окно — рабочая память агента. Она быстро забивается историей диалога, логами, содержимым файлов и данными от инструментов. На этом занятии вы учитесь смотреть, чем занят контекст, вовремя его сжимать и сохранять важные знания между сессиями.

Темы:

- из чего состоит контекст: системный промпт, инструменты, агенты, память, история и свободное место;
- почему раздутый контекст снижает точность модели;
- что засоряет контекст: длинный `CLAUDE.md`, лишние MCP-серверы, неочищенная история;
- автоматическое и ручное сжатие через `/compact`;
- команды `/context`, `/clear`, `/rewind`, `/model`;
- `/memory`, auto-memory и файл `MEMORY.md`;
- постепенное раскрытие: короткий `CLAUDE.md`, тематические правила в `.claude/rules/`, ссылки через `@path`.

**Результат**: правила, память и рабочие файлы разнесены по местам, а агент может обрабатывать новости без перегрузки контекста.

### 5. MCP и внешние системы

Материалы: [презентация](05-mcp/automatica-ccb-s1-e5.pdf), упражнения [01](05-mcp/exercise/01-mcp-architecture.pdf), [02](05-mcp/exercise/02-tavily-search.pdf), [03](05-mcp/exercise/03-digest-first-publish.pdf), [04](05-mcp/exercise/04-group-mcp-challenge.pdf).

MCP, или Model Context Protocol, даёт Claude Code доступ к внешним сервисам: GitHub, поиску, базам данных, Figma, Slack и другим инструментам. На занятии вы разбираете, как устроены MCP-серверы, как их подключать и где проходят границы безопасности.

Темы:

- архитектура MCP: хост, клиент и сервер;
- что MCP отдаёт агенту: инструменты, ресурсы и промпты;
- типы подключений: HTTP, SSE и stdio;
- команда `claude mcp add` и настройка через `.mcp.json`;
- области видимости: local, project и user;
- хранение ключей в переменных окружения;
- поиск серверов в PulseMCP, mcp.so и MCP Registry;
- MCP Inspector для отладки инструментов и параметров.

**Результат**: Claude Code умеет искать новости, работать с внешними сервисами и готовить первый выпуск дайджеста.

### 6. Навыки и автоматизация

Материалы: [презентация](06-skills-hooks/automatica-ccb-s1-e6.pdf), упражнения [01](06-skills-hooks/exercise/01-custom-skills.pdf), [02](06-skills-hooks/exercise/02-hooks-setup.pdf), [03](06-skills-hooks/exercise/03-tdd-workflow.pdf), [04](06-skills-hooks/exercise/04-digest-skill.pdf).

Вы превращаете повторяющиеся команды в навыки, хуки и проверки. Навык описывает как выполнять задачу. Хук запускается в нужный момент работы агента.

Темы:

- пользовательские навыки в `.claude/skills/`;
- YAML-шапка навыка: доступные инструменты, модель, изоляция контекста, хуки;
- аргументы через `$ARGUMENTS` и повторный запуск через `/loop`;
- хуки PreToolUse, PostToolUse, Stop и другие события;
- типы хуков: shell-команды, HTTP-запросы, LLM-проверки;
- блокировка опасных действий, автоформатирование и запуск тестов;
- TDD-цикл `RED -> GREEN -> REFACTOR`.

**Результат**: навыки для выпуска дайджеста и генерации обложек, плюс хук, который проверяет качество статьи.

### 7. Агенты и оркестрация

Материалы: [презентация](07-agents/automatica-ccb-s1-e7.pdf), упражнения [01](07-agents/exercise/01-sub-agents.pdf), [02](07-agents/exercise/02-agent-teams.pdf), [03](07-agents/exercise/03-worktrees.pdf), [04](07-agents/exercise/04-digest-parallel-agents.pdf).

Вы переходите от одного агента к нескольким: один исследует, другой пишет, третий готовит обложки, четвёртый собирает страницы. Чтобы они не мешали друг другу, у каждого есть своя задача, свой контекст и, при необходимости, отдельная рабочая копия кода.

Темы:

- встроенные субагенты Explore, Plan и General-purpose;
- собственные агенты в `.claude/agents/`;
- отдельное контекстное окно для каждого субагента;
- фоновые задачи и команда `/tasks`;
- Agent Teams: общий список задач и обмен сообщениями;
- статусы задач: pending, in_progress, completed;
- `claude --worktree` для изоляции изменений в разных рабочих копиях.

**Результат**: несколько агентов параллельно ищут новости, пишут текст, готовят обложки и собирают страницы дайджеста.

### 8. Agent Harness

Материалы: [презентация](08-harness/automatica-ccb-s1-e8.pdf), упражнения [01](08-harness/exercise/01-debugging.pdf), [02](08-harness/exercise/02-headless-mode.pdf), [03](08-harness/exercise/03-sdk-intro.pdf), [04](08-harness/exercise/04-digest-full-pipeline.pdf).

Agent Harness — это обвязка вокруг модели: правила, инструменты, доступы, навыки, хуки, агенты и проверки. На финальном занятии вы собираете эти части в один рабочий цикл и учитесь отлаживать его, когда агент застревает или чинит одно, но ломает другое.

Темы:

- из чего состоит Agent Harness;
- почему агенту лучше давать только нужные инструменты;
- линтеры и тесты как автоматическая проверка правок;
- визуальная проверка через скриншоты и Playwright MCP;
- проверка результата другой моделью;
- отладка через логи, стек-трейсы и воспроизводимые ошибки;
- запуск Claude Code в `headless`-режиме для скриптов и CI/CD;
- GitHub Actions для issues, pull requests и комментариев;
- SDK: `query()`, `ClaudeSDKClient` и собственные инструменты.

**Результат**: полный цикл AI-дайджеста — поиск тем, подготовка текста, генерация графики, проверка, коммит и публикация.

## Кому подойдёт курс

- Разработчикам, которые хотят использовать Claude Code для постоянной работы.
- Техническим специалистам, которые уверенно работают в терминале и хотят автоматизировать рутинные задачи.
- Пользователям Claude Code, Cursor и похожих инструментов, которым нужен проверяемый результат, углубление знаний.
- Тем, кто хочет разобраться в MCP, хуках, навыках, субагентах и агентных пайплайнах на практическом проекте.

Курс не получится пройти пассивно: нужно выполнять упражнения, запускать команды и собирать проект руками.

## Как проходить

1. Идите по папкам последовательно: `01-base`, `02-prompting`, ..., `08-harness`.
2. В каждой папке сначала откройте презентацию занятия.
3. Затем выполните PDF-упражнения из `exercise/` по порядку.
4. После каждой ступени фиксируйте изменения в Git.
5. Не складывайте все правила в один `CLAUDE.md`: по мере курса переносите их в память, отдельные файлы, навыки и хуки.

## Требования

- Компьютер с macOS, Windows или Linux.
- VS Code или другая IDE.
- Git и базовые навыки работы в терминале.
- Доступ к Claude Code.
- Для практической части понадобятся API-ключи Tavily, Replicate, GitHub и Vercel.
